# MrCarlsama短视频获客全链路诊断私教

这是给 Codex 使用的中文短视频获客诊断 skill。

它不是普通文案助手，也不是爆款标题工具，而是短视频获客全链路诊断私教：从账号底盘、精准用户、单条内容、选题库、案例素材、发布复盘，到必要时的私域承接，按同一套获客逻辑持续校准。

默认面向中国短视频获客场景。例子和诊断口径优先贴近本地门店、装修门窗、餐饮美业、摄影摄像、教培、企业咨询、AI 交付、知识付费、老板决策、财务卡预算、家人嫌贵、员工不配合这些真实情况。

核心判断只有一个：这条内容、这个选题、这个账号动作，是否真的服务一单生意、一个精准用户、一个真实需求和一个下一步动作。

## 为什么强调全链路

短视频获客不是单点动作。

只改脚本，不知道卖什么，内容会虚。
只追选题，不知道服务谁，流量会散。
只有播放，没有案例和信任证据，咨询会弱。
只看数据，不做复盘，账号会重复犯同一个错。
只谈私域，不知道公域筛来的是什么人，承接会变成硬聊。

比如：

- 装修账号不能只讲“避坑”，还要知道客户是刚拿房、准备开工、预算被家人卡住，还是已经被增项坑过。
- 教培账号不能只讲“提分方法”，还要知道家长是焦虑升学、孩子不配合，还是之前报班没效果。
- 企业 AI 账号不能只讲工具多厉害，还要知道老板怕花了钱员工不用，财务怕预算浪费，业务负责人怕落不了地。
- 知识付费账号不能只讲“认知升级”，还要知道用户怕买完不用、怕割韭菜、怕学完还是不会做。

所以这套 skill 不是“帮你写一条视频”，而是帮你把获客链路一层一层对齐：

1. 账号底盘：先看生意，不先聊抽象定位。
2. 精准用户：用户不是标签，是处境中的人。
3. 单条内容：判断内容服务哪类用户、哪个需求、哪个下一步动作。
4. 精准选题库：没有所谓爆款选题，只有精准选题。
5. 案例库 / 素材库：把真实反馈、服务过程、用户原话变成信任证据。
6. 发布后复盘：判断吸引的人对不对，需求有没有被激活。
7. 疑难杂症处理：把一机一号、权重、发布时间、日更焦虑拉回内容质量。
8. 私域承接进阶：只在已有线索和承接入口时诊断，不强行提前设计。

这些模块不是孤立工具。缺一层，后面的判断就容易失真。

## 使用后会获得什么

持续使用这个 skill，你会得到一套越来越准的短视频获客工作台：

- 一份清楚的账号底盘：你到底卖什么、卖给谁、为什么他现在需要、看完内容要来找你做什么。
- 一套精准用户处境：不再用年龄、性别、城市这类标签糊弄自己，而是说清用户现在卡在哪里。
- 一个可维护的选题库：每条选题都有需求、解决方向、服务人群、内容类型和状态。
- 一套内容诊断标准：知道一条内容是跑偏、能发测试，还是只需要改一层。
- 一个案例库和素材库：把客户原话、反馈、截图、服务过程沉淀成信任资产。
- 一套发布后复盘方法：不只看播放量，而是看有没有吸引对的人、带来对的咨询。
- 一个常见问题处理口径：不被权重、养号、发布时间这些问题拖走执行力。
- 一个越用越准的知识库：但任何写入都必须经过用户确认。

## 日常怎么使用

把它当成短视频获客私教，不要只在“写文案”时才用。

常见用法：

- 起号 / 换方向：说“使用 MrCarlsama短视频获客全链路诊断私教，开始初始化诊断。”
- 发内容前：丢一条选题、脚本、门店套餐介绍、课程介绍、服务介绍或免费资料介绍，让它判断框架是否跑偏。
- 想选题时：让它基于你的账号底盘、用户处境和客户语言建设精准选题库。
- 有客户反馈时：把咨询记录、评论、私信、截图、成交异议放进来，判断是否值得进入案例库或素材库。
- 发完内容后：把播放、完播、评论、私信、问价、加微信、预约、成交情况丢进来，做发布后复盘。
- 执行焦虑时：问一机一号、权重、发布时间、日更、隔日更等问题，让它把注意力拉回内容本身。
- 有线索之后：再进入私域承接进阶，诊断私信、微信、社群和成交承接。

日常使用的关键不是一次问完所有问题，而是每次只处理当前最影响获客的一层。

## 初始化会生成什么目录

第一次初始化时，不应该在根目录平铺生成几个 md 文件。正确做法是建立一个固定的全链路诊断项目骨架。

核心结构：

```text
00_总入口.md
01_账号底盘/
02_用户处境/
03_单条内容诊断/
04_精准选题库/
05_案例素材库/
06_发布复盘/
07_疑难杂症/
08_私域承接_进阶/
99_系统记录/
```

完整目录规则见：

```text
skills/aka-private-shortvideo-acquisition-diagnosis/references/project-scaffold.md
```

注意：创建目录和写入业务结论是两件事。目录可以作为项目骨架创建；卖什么、用户是谁、内容边界、客户语言这些业务结论，必须等用户确认后再写入。

进入初始化诊断前，AI 必须先读取 `project-scaffold.md`，再读取 `initialization.md`。不能只读初始化问答规则后临时发挥目录结构。

所有写入的 md 都要人类可读。也就是说，打开文件的人应该能看懂这页管什么、当前结论是什么、依据是什么、下一步做什么，而不是只看到一堆 AI 才方便解析的字段。

## 给其他 AI / Agent 的使用说明

如果你是另一个 AI，进入这个仓库后不要从 README 直接猜流程。先读：

1. `skills/aka-private-shortvideo-acquisition-diagnosis/SKILL.md`
2. 根据用户任务，再读对应的 `references/*.md`

路由规则：

- 用户要重新起号、换方向、问定位：读 `references/initialization.md`
- 用户丢选题、脚本、产品介绍、服务介绍、免费资料介绍：读 `references/content-diagnosis.md`
- 用户要建设选题库、热点选题、需求池：读 `references/topic-bank.md`
- 用户给客户故事、咨询记录、用户反馈、截图：读 `references/case-material-bank.md`
- 用户给发布后数据、评论、私信、成交情况：读 `references/review-diagnosis.md`
- 用户问一机一号、养号、权重、发布时间、日更：读 `references/common-qa.md`
- 用户已有线索，要诊断私信、微信、社群、成交承接：读 `references/private-domain-advanced.md`

## 正式开始使用

在用户的自媒体知识库或项目目录里启动，不要在这个 skill 仓库里替用户沉淀账号资料。

推荐用户这样发起：

```text
进入这个目录，使用 MrCarlsama短视频获客全链路诊断私教，开始初始化诊断。
```

初始化时，AI 必须先输出“本次私教短视频获客核心理念”，再开始提问。第一轮优先问清：

1. 用户现在最想靠短视频卖出去的具体东西。
2. 这一单多少钱，谁掏钱，谁使用，谁可能反对。
3. 用户为什么现在非解决不可。

不要先问“你的定位是什么”。按这套方法，先看生意。

## 使用铁律

- 未经用户确认，不写入用户知识库。
- 准备写入前，必须说明：写进哪里、写什么、为什么值得写。
- 写入的 md 不仅 AI 要能看懂，人也要能直接看懂。
- 内容诊断只判断框架和获客链路，不默认代写完整文案。
- 没有所谓爆款选题，只有精准选题。
- 用户不是标签，是处境中的人。
- 例子优先用中国实际情况，不默认套硅谷、海外软件、跨境电商、增长漏斗这类离用户很远的案例。
- 不用“抓手”这类黑话，直接说“内容切入口”“客户语言”“用户为什么停下来”。
