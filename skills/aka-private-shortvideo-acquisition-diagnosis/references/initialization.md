# 初始化诊断流程

当用户要重新起号、换方向、重做内容底盘，或者问“我该怎么定位”时，使用这个流程。

## 目标

先把账号底盘问清楚，再诊断内容：

- 卖什么。
- 谁掏钱，谁使用，谁影响决策。
- 用户为什么现在需要。
- 内容应该服务哪些真实处境。
- 账号不应该吸引谁。

初始化不是在根目录平铺生成几个 md 文件。第一次初始化项目时，要使用 `project-scaffold.md` 里的固定目录骨架，让后续账号底盘、用户处境、内容诊断、选题库、案例素材、复盘和私域承接各归各位。

## 开场

不要先问“你的定位是什么”。

执行初始化时，必须先输出一段“本次私教短视频获客核心理念”，再开始提问。目的不是讲课，而是让用户先认同这套诊断标准，知道自己接下来为什么要回答这些问题。

开场顺序：

1. 说明已经进入初始化诊断。
2. 输出核心理念。
3. 再问第一轮问题。

核心理念推荐模板：

```text
我用 MrCarlsama短视频获客全链路诊断私教 进入的是【初始化诊断】。

先把这次私教的核心理念说清楚：

1. 先不聊“定位”。定位是外壳，生意才是底盘。你先要说清楚：你想通过短视频卖出哪一单。
2. 短视频获客不是多发内容，也不是追所谓爆款，而是让每条内容服务一类精准用户。
3. 流量变贵、变细、变精准以后，真正值钱的是：你知道这条视频让谁停下来，让他理解什么，为什么信你，下一步为什么来咨询或购买。
4. 用户不是标签。不是年龄、性别、城市、收入，而是处境：他现在卡在哪里，为什么现在必须解决，解决不了会损失什么。
5. 所以初始化不是让你想一个漂亮人设，而是先把一单生意、一个精准用户、一个真实需求、一个下一步动作说清楚。

这套流程不会要求你一开始就完美。先把底盘问清楚，第一批内容只需要能测试精准用户，不需要一次做成最终版本。
```

随后再说：

> 先不聊定位。按这套方法，先看生意。你想通过内容先卖哪一单？

## 问题

分轮追问。用户已经清楚时，不要机械地把所有问题一次问完。

提问时的例子要贴近中国实际情况。优先用本地门店、装修门窗、餐饮、美业、摄影摄像、教培、企业咨询、AI 交付、老板/财务/合伙人/家人/员工这些真实决策关系，不要默认用西方商业案例。

1. 你现在最想靠内容卖出去的，是哪一个东西？
   - 例子：门店套餐、装修服务、摄影摄像、企业 AI 咨询、老板培训、课程、训练营、模板包、诊断、落地交付项目。
   - 如果用户回答“IP / 影响力 / 账号”，要纠正：那是外壳，不是生意。

2. 这一单多少钱？把付款决策链说清楚：谁掏钱，谁使用，谁可能反对或影响决定？
   - 这一题实际问用户时，必须带例子和回答格式。不要只问“谁掏钱，谁使用”，用户会觉得虚。
   - 不要只写“客户”。很多生意不是一个人决定，也不是掏钱的人亲自使用。
   - 例子必须先分成 3 个大方向：实体、服务、知识付费。不要散着列行业，用户会抓不住。
   - 让用户照这个格式回答：客单价 / 谁掏钱 / 谁使用 / 谁可能反对 / 反对理由。
   - 推荐问法：

     > 第二题别只写“客户”。你按这个格式回：客单价多少；谁真正掏钱；谁实际使用；谁可能拦一下；他为什么拦。
     >
     > 可以先按 3 类想：
     >
     > 实体：比如装修、门窗、家电、服装、餐饮。可能是夫妻或房主掏钱，全家使用，反对理由可能是贵、怕踩坑、怕返工、怕不好看。
     >
     > 服务：比如企业 AI 交付、摄影、设计、装修服务、咨询顾问。可能是老板或负责人掏钱，员工或客户使用，财务、技术、合伙人可能反对，理由是预算、风险、配合成本、看不见效果。
     >
     > 知识付费：比如 AI 小课、训练营、陪跑、模板包。可能是本人掏钱本人用，也可能是公司掏钱团队用；反对者可能是自己、家人、老板，理由是怕买了不用、怕割韭菜、怕没时间学、怕学完不会落地。

   - 实体：装修/门窗/家电/服装/餐饮，通常要看购买者、使用者、家庭或同伴影响者。
   - 服务：企业 AI 交付/摄影/设计/咨询/顾问，通常要看付款负责人、实际配合者、内部反对者。
   - 知识付费：课程/训练营/陪跑/模板包/诊断，通常要看本人决策、时间成本、信任顾虑、学完能不能落地。
   - 如果暂时不知道谁反对，可以写“我猜可能是……”，不要空着。

3. 客户为什么现在非解决不可？
   - “有用”不够。人会行动，是因为现在有麻烦、压力、截止时间、损失或机会。

4. 用户现在过的是什么日子？钱从哪来，信息从哪来？
   - 不要依赖性别、年龄、城市、收入这些标签。
   - 要看人生阶段、收入结构、信息来源。

5. 他最怕什么损失？
   - 怕白花钱、怕选错、怕丢脸、怕耽误、怕效果不稳定、怕损失钱、怕失去信任。

6. 他凭什么信你？
   - 案例、前后对比、过程记录、判断标准、反馈、截图、服务记录、个人经历。

7. 看完内容后，他应该带着什么来找你？
   - 一个问题、一份材料、一张截图、一个预算、一个处境、一版草稿、一个决策问题，或者明确的咨询意图。

8. 第一批内容先推动用户到哪一步？
   - 知道你、理解你、信任你、来咨询、下单。

## 输出

```text
【账号底盘】
卖什么：
客单价：
谁掏钱：
谁使用：
谁可能反对/影响决策：

【用户处境】
用户现在的生活/业务状态：
当前最急的问题：
为什么现在重要：
最怕的损失：
信息来源/语言习惯：

【内容边界】
应该吸引谁：
不应该吸引谁：
容易跑偏到哪里：

【人设信任】
用户需要信任我什么：
我能提供的证据：

【第一批内容方向】
可测试用户需求：
可测试内容类型：
下一步动作：
```

如果输出里有可复用的账号底盘信息，必须先询问用户是否写入知识库。

## 初始化写入规则

用户回答完初始化问题后，先输出第一版【账号底盘】，不要立刻写文件。

如果用户确认写入，按 `project-scaffold.md` 的固定目录写：

```text
00_总入口.md
01_账号底盘/生意底盘.md
01_账号底盘/产品与成交动作.md
01_账号底盘/内容边界.md
02_用户处境/用户处境库.md
02_用户处境/高意愿场景.md
02_用户处境/客户语言库.md
99_系统记录/log.md
99_系统记录/写入确认记录.md
```

不要生成图省事的平铺文件，例如：

```text
账号底盘.md
第一批内容方向.md
内容边界.md
用户困境_xxx.md
```

这种结构短期看方便，后面做选题、案例、复盘和私域承接时会乱。
